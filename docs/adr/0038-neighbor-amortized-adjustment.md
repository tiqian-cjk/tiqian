# ADR 0038: 邻行均摊的行长调整(NeighborAmortizedAdjustment)

- Status: Accepted
- Date: 2026-07-06
- Refines: [ADR 0031](0031-line-adjustment-direction.md)(方向选择不变,本 ADR 只改**分布**)

## Context

ADR 0031 修订把方向定为「先推入、后推出」,但 dogfood(知乎++ 真机)显示观感问题
并未消失。复盘发现真正的病根在**调整的分布**,而不是方向:

- lookahead 的行代价是**线性**的(`差额 × 权重`):一行调 56px 与两行各调 28px
  总分相同,优化器没有任何均摊动机;
- 代价**不按密度归一**:56px 摊在 20 个字距上(每距 2.8px,不可察觉)与摊在
  5 个字距上(每距 11px,极刺眼)同分;
- **没有邻行项**:一行挤得极紧、下一行拉得极松,零惩罚;
- 填充推入(fill PushIn)是**二值**的——能整块压进来就压,专门制造
  「一行大调、邻行不调」的密度断崖。

外部的字体排印评审意见所指的「算法不细致」正是这一层:减少调整应当理解为
**减少可察觉的调整**,即把量均摊到临近行,而非减少调整的**次数**
(要么不调、要么一次调个大的)。

## Decision

行代价改为**按字距密度计价的凸函数 + 邻行差项**(TeX `adj_demerits` 同思路):

```
d_i      = 该行调整量 / max(1, 该行 CJK 字距数)     // 有符号:拉伸 +,压缩 −
fill_i   = (d_i² / d_ref) × Wf                      // 凸:两行各半 < 一行全量
adj_i    = ((d_i − d_{i−1})² / d_ref) × Wa          // 邻行密度落差
badness  = fill_i + adj_i + 修复罚分(不变)
```

- `d_ref` = 每字距拉伸上限(`maxCjkStretchPerGap`),把平方项归一回 px 量纲;
- 末行与 MandatoryBreak 行的 `d = 0`(读者预期段末自然收束,不参与均摊,
  邻行差项也不跨强制断行);
- `Wf = Wa = raggednessWeight`(初值,按语料再调);
- 无字距行(纯西文/单字)按 justify 后状态计价:亏空填不进字距,保持
  **线性** ragged 代价(也正是 standalone breaker 合同的旧行为);
- `SingleClusterLinePenalty`:非末行孤字行显式罚分(= LeaveRagged 罚分),
  只作最后手段;
- 填充推入的接受条件从「能压就压」改为:**引入的压缩密度不超过其
  消除的拉伸密度**(逐字距归一;`overflow ≤ 0` 时纯赚必收)。填充是
  级联波——下一行会再从它的下一行回填,故亏空不在本对内计价;推入
  依旧优先(方向不变),但不再以「一行大压」换「一处小差」。

## Consequences

- 断行选择会系统性偏向「相邻两行各担一点」而非「一行全担」;golden 普遍
  变化,`TIQIAN_UPDATE_GOLDEN=1` 重生成并逐行 review 邻行密度;
- `PushOutOnly` 行为不变(从不推入);方向档位语义不变;
- dump 的 `line:` 注记补充每行字距数与密度,保持可解释。

## Alternatives considered

- **全局最优(Knuth-Plass 全段 DP)**:效果上界,但成本与现有
  greedy+lookahead 架构差异过大;先用窗口内均摊吃掉主要观感收益,
  不排除后续演进。
- **只加邻行项、保持线性 fill**:线性项仍无均摊动机,一行全担 + 邻行差
  两项打架,调参脆弱。凸 fill 是均摊的根,邻行项是补充。
- **在 Justifier(行内分配)层做**:行内各档分配已按 CLREQ 档序,均摊是
  **跨行**问题,只能在断行/填充决策层解决。
